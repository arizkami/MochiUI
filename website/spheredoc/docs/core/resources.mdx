# ResourceManager

**Header**: `include/core/ResourceManager.hpp`

Singleton registry for embedded binary resources. Resources are accessed via the `res://name` URI scheme used by `SPHXImage`, `SPHXSvg`, and other asset loaders.

## Struct

```cpp
struct ResourceData {
    const uint8_t* data;
    size_t size;
};
```

## Class Reference

```cpp
class ResourceManager {
public:
    static ResourceManager& getInstance();

    void registerResource(const std::string& path,
                          const uint8_t* data,
                          size_t size);

    ResourceData getResource(const std::string& path);
    std::string  getResourceString(const std::string& path);
};
```

### `registerResource(path, data, size)`

Registers a raw memory block under the given URI path. The `data` pointer must remain valid for the lifetime of the application (use `static` arrays or generated headers).

### `getResource(path)`

Returns a `ResourceData` pointing to the registered memory. Returns `{nullptr, 0}` if not found.

### `getResourceString(path)`

Convenience wrapper that constructs an `std::string` from the resource bytes.

## Example

```cpp
// Generated header from gen_resources.py
#include "AppResources.hpp"

// Register at startup
auto& rm = ResourceManager::getInstance();
rm.registerResource("res://icon.svg",
    AppResources::icon_svg,
    AppResources::icon_svg_size);

// Use anywhere via URI
auto svg = SPHXSvg::fromResource("res://icon.svg");
```
